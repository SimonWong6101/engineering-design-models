Tool Path Customization - Entry/Exit

Often the creation of a tool path to profile a feature or cut a surface is only the beginning for the NC Programmer. Next he must customize the tool path, capturing precisely

  • how and where the tool should approach the material,
  • how it should exit,
  • how many times it should go over the material and
  • how much material should be removed with each pass,
and any number of other options. Pro/MFG provides these and many more tools for customizing a tool path, for both optimum cutting motion and for minimizing machining time .




In the following examples, a bull nose cutter profiles an electrode for a finish cut. The programmer has selected  the edges around the part and let the system determine the start and end point.
 

As you can see, the tool enters in the center of the lower cutaway. For our first customization, let's simply change the start and end point to be on the side. We can put the start point anywhere along the path.
 

This is better, with the start and end point being at the start of the right flat. Still, we probably don't want the tool plunging against the part. What we might want to do is to extend the start and end points off of the part.
 

Alternatively, maybe we want to have the tool use a normal move straight into part or it can be an arc lead in and lead out.
 


Now, maybe there is too much material left for this cut to be machined in one pass. The programmer has the option of adding cuts in the Z direction or in the profile direction, or both.
 

Next we'll look at some of the customization available with cavity roughing operations. In this example, we have a forging die cutout that is rough machined using a 3/8" flat end mill. At first, with no entry selection, the system begins its spiral cutting motions by plunging at roughly the center of each level.
 

Many NC Programmers would prefer to first drill a hole, then approach along that hole axis to each appropriate z depth, machine out that depth, return to the hole, plunge again, and repeat the process. We can do this by first adding a drilling sequence and then simply telling Pro/MFG to use the same axis as its start and end point.
 


Alternatively, a helical approach could be used. Here the programmer inputs a desired diameter and ramp angle. Instead of plunging, the system will now move the in a helical path when entering each Z-level.
