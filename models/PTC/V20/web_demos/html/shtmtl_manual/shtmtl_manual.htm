Pro/NC-SHEETMETAL Manual Programming



Several new enhancements have been added to the manual side of NC programming. Automated roughing and profiling for punching, new dialog boxes to futher increase the efficiency of customizing contouring tool paths, new toolpath optimization tools, and new workcell management tools are available.

Rough and Profile Punching
For manufacturing parts that have large internal voids, Pro/NC-SHEETMETAL can "destroy" the area by filling the boundary of the void with hits. This sequence type is called "Nibble Area". To improve edge quality in these areas and reduce machining time, it is often desirable to punch out the area using 2 different tools. By setting the sequence parameter "Scan-Type" to "Rough and Profile", and defining the "second tool" option, Pro/NC-SHEETMETAL will rough the area with the first tool and then profile the edges of the boundary with the second tool. Area roughing can also be done with a tool loaded at an angle.



Laser-Flame Contouring
Corner Conditions
For lasers and plasma tables, an important consideration is how the corners of the part are treated. Many times it is desirable to cut past a corner, loop around and engage the next edge. Pro/NC-SHEETMETAL does this by giving the user full control over what kind of corner condition to automatically apply to the toolpath. Sizes and types of the moves can be preset in parameter files or specified "on the fly" in dialog boxes.

Approach and Exit Moves
Pro/NC-SHEETMETAL can automatically place a variety of approach and exit moves. These moves can be linear such as approching from an angle at a distance, or radial tangent arc lead in. Optionally the user can decide to start at the beginning or in the middle of an edge. Sizes of the moves can be preset in parameter files or specified "on the fly" in dialog boxes.



Toolpath Optimization
To reduce machine run times and produce the most efficient tool motions Pro/NC-Sheetmetal provides the user complete control on ordering the tool usage, minimizing tool changes, and creating "shortest path" programs. Notice in the example, the "as programmed" tool path is constantly changing tools and completes one part before moving to the next. The "optimized" tool path is much more efficient, completely using one tool before moving to the next and produces a tool path that takes less time to execute.



Workcell Management

To streamline the workflow of the NC programmer, a new Turret Management Dialog Box has been added. This allows the user to create and manage the tool library, get a quick glimpse of which tools are actually in use for the program, check holder sizes and indexable stations. Tools can be retrieved from the library or created on the fly.
